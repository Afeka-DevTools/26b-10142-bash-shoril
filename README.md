שמות חברי הקבוצה: שון ורחובסקי, עילאי מיכלובסקי, אור אלמוג 


על מנת להשתמש בסקריפטים של הפרויקט שלנו צריך מערכת הפעלה לינוקס. במידה ואתם משתמשים במערכת של המכללה יש להתחבר לvlab של devtools. שם המשתמש והסיסמה הם גם devtools. לאחר מכן לפתוח את הטרמינל ולכתוב את הפקודה הבאה:
git clone https://github.com/Afeka-DevTools/26b-10142-bash-shoril.git
עכשיו צריך לפתוח את התיקייה הזאת. את זה נעשה על ידי שימוש בפקודות הבאות:
cd 26b-10142-bash-shoril.git
cd scripts
עכשיו אנחנו נמצאים בתוך התיקייה של הסקריפטים שלנו. על מנת להשתמש באחד מהם צריך להוסיף לא הרשאה של execute (X) את זה נעשה על ידי הפקודה הבאה:
sudo chmod +x ["השם של הסריפט אותו אתם רוצים להריץ"]
במקום הסוגריים אתם מכניסים את השם של הקובץ אותו אתם רוצים להריץ לא לשכוח את הסיומת.
כנראה שבשלב הזה תתבקשו להקיש סיסמה. במידה ואתם עובדים במערכת הפעלה שלכם יש להקיש את הסיסמה שלכם למשתמש. ואם אתם עובדים בvlab הסיסמה היא devtools.
עכשיו ניתן לבדוק שאכן הוספנו את ההרשאה להרצה על ידי הרצה של הפקודה ls -l שם הקובץ יצבע בצבע ירוק ואם לא אתם תראו שיש בצד ימין ליד הקובץ את האות x.
עכשיו אתם מוכנים להריץ את הסקריפט הזה. על מנת לעשות את זה יש להקיש את הפקודה הבאה:
./"The name of the script you want to run"
במקום כל המירכאות אתם רושמים את השם של הסקריפט שאתם רוצים להריץ כמובן שהוא ירוץ רק אם כבר הוספתם לו את ההרשאה לexecute (X).

שון בחר לעשות את הסקריפטים הבאים:
1. check_network.sh - בודק את החיבור שלכם לאינטרנט אין צורך להזין שום קלט רק להריץ את הפקודה כמו שרשום למעלה.
2. password_generator.sh - מכין סיסמה בת 10 תווים עם לפחות אות אחת גדולה, מספר אחד וסימן מיוחד אחד. אין תורך בשום פלט מהמשתמש רק הרצה לפי ההוראות למעלה.
3. timer.sh - טיימר שסופר זמן לפי מה שהמשתמש מכניס. יש צורך להכניס שעות,דקות ושניות בפורמט הבא:
   HH:MM:SS כלומר 2 ספרות ראשונות יהיו השעות אחרי זה נקודותיים ואז כמות הדקות ואז נקודותיים ואז את השניות. 
4. user_info.sh - אין תורך להכניס שום ערכים רק להריץ את הפקודה כראוי.
5. weather.sh - יש צורך להכניס את העיר בשפה האנגלית שרוצים לבדוק לה את מזג האוויר.

עילאי בחר לעשות את הסקריפטים הבאים:
1. backup_directory.sh - מגבה תכולה של תיקיה לקובץ מכווץ (tar.gz). חובה להכניס כקלט את נתיב התיקיה שרוצים לגבות לאחר פקודת ההרצה.
2. delete_old_files.sh - מוחק קבצים הישנים יותר ממספר ימים שתגדירו מתיקיה מסוימת. חובה להכניס כקלט את נתיב התיקיה ולאחריו את מספר הימים.
3. disk_usage.sh - מציג את שטח הדיסק (המשקל) שתופסת כל תיקיה. אפשר להכניס נתיב לתיקיה כקלט, או להריץ ללא קלט והוא יבדוק את התיקיה הנוכחית.
4. count_elements.sh - מסכם את מספר הקבצים הרגילים, התיקיות והלינקים בתיקיה מסוימת. גם כאן, אפשר להכניס נתיב כקלט, או להריץ ללא קלט עבור התיקיה הנוכחית.
5. system_uptime.sh - מציג את זמן הפעילות של המערכת (Uptime). אין צורך להזין שום קלט, רק להריץ את הפקודה כראוי.

אור בחר לעשות את הסקריפטים הבאים:

1. סריקת פורטים פתוחים (port_scanner.sh)
תיאור: הסקריפט סורק אילו פורטים פתוחים עבור כתובת IP מסוימת. הסקריפט מוודא שמותקן nmap ומתקין אותו אוטומטית במידת הצורך.

הכנה להרצה (מתן הרשאות):
chmod +x port_scanner.sh

הרצת הסקריפט (אפשרות א' - הזנת כתובת ה-IP בשורת הפקודה):
./port_scanner.sh 127.0.0.1

הרצת הסקריפט (אפשרות ב' - הרצה ללא קלט מראש, הסקריפט יבקש להקליד):
./port_scanner.sh

2. הצגת סטטוס סוללה (battery_status.sh)
תיאור: הסקריפט בודק את מצב הסוללה של המחשב. הסקריפט מוודא שמותקן הכלי acpi (ומתקין אותו במידת הצורך). בנוסף, הסקריפט יודע לזהות האם הוא רץ על מכונה וירטואלית/נייח ללא סוללה ומדפיס הודעה בהתאם.

הכנה להרצה (מתן הרשאות):
chmod +x battery_status.sh

הרצת הסקריפט (אין צורך בקלט):
./battery_status.sh

3. הריגת תהליכים לפי שם (kill_process.sh)
תיאור: הסקריפט מסיים (הורג) את כל התהליכים שרצים תחת שם שהוגדר. הסקריפט מוודא שחבילת psmisc (המכילה את פקודת killall) מותקנת.

הכנה להרצה (מתן הרשאות):
chmod +x kill_process.sh

הרצת הסקריפט (אפשרות א' - העברת שם התהליך כקלט בשורת הפקודה):
./kill_process.sh [process_name]
דוגמה: ./kill_process.sh firefox

הרצת הסקריפט (אפשרות ב' - הרצה ללא קלט מראש, הסקריפט יבקש להקליד):
./kill_process.sh

4. תזכורת על המסך (reminder.sh)
תיאור: הסקריפט ממתין פרק זמן מוגדר מראש ואז מקפיץ חלונית התראת מערכת (Pop-up Notification) מחוץ לטרמינל. הסקריפט מוודא שחבילת libnotify-bin (המכילה את פקודת notify-send) מותקנת במערכת.

הכנה להרצה (מתן הרשאות):
chmod +x reminder.sh

הרצת הסקריפט (אפשרות א' - הזנת הזמן וההודעה בשורת הפקודה):
./reminder.sh [time] [message]
דוגמה: ./reminder.sh 1m Time to drink water

הרצת הסקריפט (אפשרות ב' - הרצה ללא קלט מראש, הסקריפט יבקש להקליד):
./reminder.sh

5. השוואה בין שני קבצים (compare_files.sh)
תיאור: הסקריפט משווה בין התוכן של שני קבצים שונים. כדי להציג את ההבדלים בצורה ברורה וצבעונית, הסקריפט מוודא שמותקנת התוכנה colordiff ומתקין אותה במידת הצורך. בנוסף, הסקריפט בודק שהקבצים שהוזנו אכן קיימים לפני ניסיון ההשוואה.

הכנה להרצה (מתן הרשאות):
chmod +x compare_files.sh

הרצת הסקריפט (אפשרות א' - העברת נתיבי הקבצים בשורת הפקודה):
./compare_files.sh [file1] [file2]
